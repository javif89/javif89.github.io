Simple GitHub

A simple, single page reference for common github commands.

Let's Begin:

*Disclaimer this page assumes you already have knowledge on VCS and that you have already installed GitHub

Cloning a repository

The process is actually very straight forward, just go on your command line in the colder where you want to clone the repository and use the command:

git clone <source-url-here>

Pushing changes to GitHub

The process of pushing changes involves only a few steps.

Step 1: Adding files to track

You first need to add files to be tracked by GitHub. This tells GitHub that these are the files you're working on and the ones it should upload once you make the push.

To add a specific file use:

git add <file-name-here>

To add ALL the files use:

git add .

Step 2: Commiting Changes

git commit -m "insert documentation message explaining what you changed here"

Step 3: Pushing to the Repository

git push

When you push it might ask for your GitHub username and password if you have not done a push before. Just put them in and it will work.

Pulling changes from GitHub

This process should be done before beginning your work. However it should not be done in the middle of your work before pushing your own changes as you might lose your work

git pull

This code will update your folder's files with ones from the GitHub repository.
